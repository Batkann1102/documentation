## Зорилго

Шүүмжид «фреймворкийн performance, scalability, memory үнэлгээ дутмаг» гэж дурдсан тул
суурь хэмжилт хийдэг benchmark module-ийг шууд төсөл дотор оруулсан. Тус module нь:

- Warmup phase
- p50, p95, min, max-ийг бодолцсон тайлан
- JSON ба Markdown форматтай үр дүн

бүхий стандарт benchmark гүйцэтгэдэг.

## CLI ашиглалт

```bash
mvn -DskipTests package
java -cp target/classes:target/test-classes \
     mn.edu.num.benchmark.BenchmarkRunner \
     --package mn.edu.num.app.scope \
     --warmup 200 \
     --iterations 1000 \
     --out target/benchmarks
```

| Параметр | Default | Тайлбар |
| --- | --- | --- |
| `--package` | `mn.edu.num.app` | Scan хийх root package |
| `--warmup` | `200` | JIT уях warmup тоо |
| `--iterations` | `1000` | Хэмжих iteration |
| `--out` | `target/benchmarks` | JSON/MD гарах хавтас |

## Devin VM-ийн жишээ үр дүн

`mn.edu.num.app.scope` (4 bean), warmup=100, iterations=500 — Devin VM (нэг машин,
GraalVM-гүй):

| Name | Iterations | Mean (µs) | P50 (µs) | P95 (µs) | Min (µs) | Max (µs) | Notes |
|------|-----------:|----------:|---------:|---------:|---------:|---------:|-------|
| startup-eager | 20 | 1681.27 | 876.12 | 7484.38 | 778.82 | 7484.38 | lazyInit=false |
| startup-lazy | 20 | 1355.09 | 553.14 | 6108.37 | 455.14 | 6108.37 | lazyInit=true |
| singleton-lookup | 500 | 1.26 | 0.95 | 1.12 | 0.80 | 44.10 | Cached singleton lookup |

Энэ файлуудыг repo дотор `docs/benchmarks/results.{md,json}` хэлбэрээр хадгалсан.

<Tip>
  Бодит апп дээрээ benchmark хийхдээ тэргүүлэх bean-уудтай pretty representative package
  өгөх хэрэгтэй. 4 bean-тэй жишээ дээр lookup-ийн mean нь зөвхөн direct map lookup-ийг л
  харуулах юм.
</Tip>

## Тайлан унших

`results.md` нь Markdown хүснэгт. `results.json` нь дараах бүтэцтэй:

```json
[
  {
    "name": "startup-eager",
    "iterations": 20,
    "meanNanos": 1681268,
    "p50Nanos": 876120,
    "p95Nanos": 7484379,
    "minNanos": 778820,
    "maxNanos": 7484379,
    "notes": "lazyInit=false (eager pre-instantiation)"
  }
]
```

CI-д хадгалаад түүхэн тренд ажиглах нь thesis-ийн өгөгдлийн чанарт ач холбогдолтой.

## Хязгаарлалт ба ирээдүй

- **JIT noise**: Warmup ихэсгэх тусам p95 баталгаатай ойртоно. JMH-ийн хэмжээний нарийвчлал
  биш ч, аппликейшн-түвшний ажиглалтад тохиромжтой.
- **GC pause**: Хэмжилт нь JVM хэмжих хугацаанд GC trigger хийсэн бол outliers үүснэ.
- **Memory profiling**: `BenchmarkRunner` нь зөвхөн latency-г хэмждэг. Heap profile авахдаа
  `jcmd <pid> GC.heap_info` эсвэл async-profiler-г санал болгоно.
