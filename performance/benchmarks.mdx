## Зорилго

Фреймворкийн хурд, өргөтгөх боломж, санах ойн (memory) хэрэглээг бодитоор үнэлж, баталгаажуулах зорилгоор Гүйцэтгэлийн үзүүлэлт хэмжих модулийг (BenchmarkRunner) шууд төсөлд багтаан боловсруулсан. Энэхүү модуль нь дараах боломжуудтай:

- Урьдчилсан халаалт хийх (warmup) үе шаттай.
- p50, p95, min, max гэх мэт нарийвчилсан статистик үзүүлэлтүүдийг тооцоолно.
- Хэмжилтийн үр дүнг JSON болон Markdown форматаар гаргаж өгнө.

## CLI ашиглалт

`ash
mvn -DskipTests package
java -cp target/classes:target/test-classes \
     mn.edu.num.benchmark.BenchmarkRunner \
     --package mn.edu.num.app.scope \
     --warmup 200 \
     --iterations 1000 \
     --out target/benchmarks
`

| Параметр | Анхдагч утга | Тайлбар |
| --- | --- | --- |
| --package | mn.edu.num.app | Обьектуудыг (bean) хайх үндсэн багцын нэр (package) |
| --warmup | 200 | JIT компиляторыг бэлтгэхийн тулд хэмжилт эхлэхээс өмнө гүйцэтгэх давталтын тоо |
| --iterations | 1000 | Бодитоор хэмжилт хийх давталтын тоо |
| --out | 	arget/benchmarks | Гаралтын тайлан (JSON/MD) хадгалагдах хавтасны зам |

| Name | Давталт | Дундаж хурд (µs) | P50 (µs) | P95 (µs) | Min (µs) | Max (µs) | Тайлбар |

## Тайлангуудыг уншуулах нь


esults.md нь хүний уншихад зориулагдсан Markdown хүснэгт бол 
esults.json нь доорх бүтцээр гарна:

`json
[
  {
    "name": "startup-eager",
    "iterations": 20,
    "meanNanos": 1681268,
    "p50Nanos": 876120,
    "p95Nanos": 7484379,
    "minNanos": 778820,
    "maxNanos": 7484379,
    "notes": "lazyInit=false (eager pre-instantiation)"
  }
]
`

Энэхүү файлыг CI (Continuous Integration) системтэй холбож хянах нь цаашдын хөгжүүлэлтийн хурд удааширч байгаа эсэхийг ажиглахад тустай юм.

## Алдаа магадлал болон цаашдын сайжруулалт

- **JIT компиляторын нөлөөлөл**: Урьдчилж халаах (warmup) давталтын тоог ихэсгэвэл P95 үзүүлэлт хамаагүй илүү бодитой гарна. Энэхүү систем нь JMH (Java Microbenchmark Harness) хэмжээний дээд зэргийн нарийвчлалгүй ч түвшин дундын статистик авахад хангалттай.
- **GC зогсолт (pause)**: Системийн хэмжилт дундуур JVM хог цэвэрлэгчээ (Garbage Collector) ажиллуулах үед хэт өндөр (outlier) утгууд гарч болно.
- **Санах ойн (Memory) хэмжилт**: BenchmarkRunner нь өнөөдрийн байдлаар зөвхөн гүйцэтгэх хурдыг хэмжиж байгаа. Тиймээс санах ойн зарцуулалтыг хэмжихийн тулд jcmd <pid> GC.heap_info команд эсвэл async-profiler зэрэг тусгай төхөөрөмж ашиглахыг зөвлөнө.

Бүхэл бүтэн фреймворкуудын гүйцэтгэлийг [Харьцуулалт (Comparison)](/performance/comparison) хуудаснаас үзэх боломжтой.
