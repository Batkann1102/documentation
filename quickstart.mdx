## 1. Bean class тодорхойлох

<CodeGroup>
```java UserRepository.java
package mn.edu.num.app.demo;

import mn.edu.num.annotation.Repository;

@Repository
public class UserRepository {
    public String findById(long id) {
        return "user#" + id;
    }
}
```

```java UserService.java
package mn.edu.num.app.demo;

import mn.edu.num.annotation.Autowired;
import mn.edu.num.annotation.Service;

@Service
public class UserService {
    @Autowired
    private UserRepository repo;

    public String hello(long id) {
        return "Hi, " + repo.findById(id);
    }
}
```
</CodeGroup>

## 2. `@EnableIoC` тэмдэглэгээ

```java DemoApp.java
package mn.edu.num.app.demo;

import mn.edu.num.annotation.EnableIoC;
import mn.edu.num.container.ApplicationContext;

@EnableIoC(basePackage = "mn.edu.num.app.demo")
public class DemoApp {
    public static void main(String[] args) {
        try (ApplicationContext ctx = new ApplicationContext(DemoApp.class)) {
            UserService svc = ctx.getBean(UserService.class);
            System.out.println(svc.hello(42));
        }
    }
}
```

<Tip>
  `try-with-resources`-д хийснээр контейнер хаагдахад бүх `@PreDestroy` метод дуугарна.
  Дэлгэрэнгүйг [Lifecycle](/core/lifecycle) хуудаснаас үзнэ үү.
</Tip>

## 3. Builder API ашиглан ачаалж эхлэх (startup)

`@EnableIoC`-н оронд програмаар тохируулах бол:

```java
ApplicationContext ctx = ApplicationContextBuilder.builder()
        .basePackage("mn.edu.num.app.demo")
        .lazyInit(true)
        .strictScope(true)
        .build();
```

[Builder API](/core/builder) дээр lazy init, strict scope, dependency tree HTML-аар нээх
зэрэг бүх сонголтын тайлбар бий.

## 4. Bean хайлт (lookup)

```java
UserService byType  = ctx.getBean(UserService.class);
UserService byName  = (UserService) ctx.getBean("userService");
```

`getBean(name)` дуудлагад нэр буруу байвал `BeanNotFoundException` болж "did you дундаж"
зөвлөмж буцаана. Жишээ нь:

```text
Bean олдсонгүй: 'usrSrvce'. Магадгүй та 'userService'-г санасан уу?
```

Дэлгэрэнгүйг [Алдааны мэдээлэл](/dx/error-messages) хуудаснаас үзнэ үү.
